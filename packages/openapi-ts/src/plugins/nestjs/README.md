# NestJS Plugin

Generate per-tag `ControllerMethods` type aliases from OpenAPI specs for NestJS controllers.

## Design Decisions

| Decision              | Choice                                         | Rationale                                                   |
| --------------------- | ---------------------------------------------- | ----------------------------------------------------------- |
| Output format         | Per-tag `{Tag}ControllerMethods` type aliases  | Maps 1:1 to NestJS one-controller-per-resource pattern      |
| `type` vs `interface` | `type` alias                                   | DSL produces `type` aliases; both work with `implements`    |
| Parameters            | Individual: `path`, `query`, `body`, `headers` | Maps 1:1 to `@Param()`, `@Query()`, `@Body()`, `@Headers()` |
| Parameter ordering    | Required before optional                       | Prevents TS error when optional precedes required           |
| Return type           | `Promise<OperationResponse>`                   | Controllers return values, not status-code maps             |
| Config options        | None                                           | Zero config = fastest path to value                         |
| Untagged operations   | `DefaultControllerMethods`                     | Consistent fallback                                         |
| `includeInEntry`      | `false`                                        | Users import from `nestjs.gen.ts` directly                  |

## Constraints

**`implements` requires whole-object parameter style:**

```typescript
// WORKS — whole-object style
async showPetById(@Param() path: ShowPetByIdData['path']) { ... }

// BREAKS — decomposed style (different signature than generated type)
async showPetById(@Param('petId') petId: string) { ... }
```

**Incompatible patterns:**

- `@Res()` — extra parameter breaks assignability
- `@UploadedFile()` — bypass `implements` for file upload methods
- SSE endpoints — `Observable<MessageEvent>` doesn't match `Promise<T>`

**Not generated:**

- Cookie parameters — NestJS handles cookies via `@Req()` or cookie middleware, not method params
- Error responses — NestJS handles errors via exception filters

## Edge Cases

| Case                    | Handling                                     |
| ----------------------- | -------------------------------------------- |
| No operationId          | Auto-generated by `operationToId()`          |
| Duplicate operation IDs | Numeric suffix (`result2`, `result3`)        |
| Invalid TS identifiers  | Sanitized by `sanitizeNamespaceIdentifier()` |
| No parameters           | `() => Promise<T>`                           |
| No response body (204)  | `Promise<void>`                              |
| Multiple tags           | First tag wins                               |
| Non-ASCII in tags/names | Handled                                      |
