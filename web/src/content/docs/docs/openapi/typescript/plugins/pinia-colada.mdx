# Pinia Colada

### About

[Pinia Colada](https://pinia-colada.esm.dev) is the perfect companion to Pinia to handle async state management in your Vue applications.

The Pinia Colada plugin for Hey API generates functions and query keys from your OpenAPI spec, fully compatible with SDKs, transformers, and all core features.

### Collaborators

<AuthorsList people={[dmitriyBrolnickij, joshHemphill, sebastiaanWouters]} />

## Features

- Pinia Colada v0 support
- seamless integration with `@hey-api/openapi-ts` ecosystem
- create query keys following the best practices
- type-safe query options and mutation options
- minimal learning curve thanks to extending the underlying technology

## Installation

In your [configuration](/docs/openapi/typescript/get-started), add `@pinia/colada` to your plugins and you'll be ready to generate Pinia Colada artifacts. 🎉

```js {6} title="openapi-ts.config.ts"
export default {
  input: 'hey-api/backend', // sign up at app.heyapi.dev
  output: 'src/client',
  plugins: [
    // ...other plugins
    '@pinia/colada',
  ],
};
```

:::tip
When using this plugin in a Nuxt app, prefer the [ofetch client](/docs/openapi/typescript/clients/ofetch) for universal compatibility.

The [nuxt client](/docs/openapi/typescript/clients/nuxt) is tailored for working directly with Nuxt composables (`$fetch` / `useFetch` / `useAsyncData`) and is not intended as a universal HTTP client for libraries like `@pinia/colada`.
:::

## Output

The Pinia Colada plugin will generate the following artifacts, depending on the input specification.

## Queries

Queries are generated from [query operations](/docs/openapi/typescript/configuration/parser#hooks-query-operations). The generated query functions follow the naming convention of SDK functions and by default append `Query`, e.g., `getPetByIdQuery()`.

<Tabs>
  <TabItem label="example">
```ts title="colada.gen.ts"
const query = useQuery(getPetByIdQuery, () => ({
  path: {
    petId: 1,
  },
}));
```
  </TabItem>
  <TabItem label="config">
```js {8} title="openapi-ts.config.ts"
export default {
  input: 'hey-api/backend', // sign up at app.heyapi.dev
  output: 'src/client',
  plugins: [
    // ...other plugins
    {
      name: '@pinia/colada',
      queryOptions: true,
    },
  ],
};
```
  </TabItem>
</Tabs>

You can customize the naming and casing pattern for `queryOptions` functions using the `.name` and `.case` options.

### Tags

You can include operation tags in your query keys by setting `tags` to `true`. This will make query keys larger but provides better cache invalidation capabilities.

<Tabs>
  <TabItem label="example">
```ts {8} title="colada.gen.ts"
const key = [
  {
    _id: 'getPetById',
    baseUrl: 'https://app.heyapi.dev',
    path: {
      petId: 1,
    },
    tags: ['pets', 'one', 'get'],
  },
];
```
  </TabItem>
  <TabItem label="config">
```js {9} title="openapi-ts.config.ts"
export default {
  input: 'hey-api/backend', // sign up at app.heyapi.dev
  output: 'src/client',
  plugins: [
    // ...other plugins
    {
      name: '@pinia/colada',
      queryKeys: {
        tags: true,
      },
    },
  ],
};
```
  </TabItem>
</Tabs>

### Accessing Query Keys

If you have access to the result of query options function, you can get the query key from the `key` field.

<Tabs>
  <TabItem label="example">
```ts title="colada.gen.ts"
const { key } = getPetByIdQuery({
  path: {
    petId: 1,
  },
});
```
  </TabItem>
  <TabItem label="config">
```js {8} title="openapi-ts.config.ts"
export default {
  input: 'hey-api/backend', // sign up at app.heyapi.dev
  output: 'src/client',
  plugins: [
    // ...other plugins
    {
      name: '@pinia/colada',
      queryOptions: true,
    },
  ],
};
```
  </TabItem>
</Tabs>

Alternatively, you can access the same query key by calling query key functions. The generated query key functions follow the naming convention of SDK functions and by default append `QueryKey`, e.g., `getPetByIdQueryKey()`.

<Tabs>
  <TabItem label="example">
```ts title="colada.gen.ts"
const key = getPetByIdQueryKey({
  path: {
    petId: 1,
  },
});
```
  </TabItem>
  <TabItem label="config">

  </TabItem>

## Mutations

Mutations are generated from [mutation operations](/docs/openapi/typescript/configuration/parser#hooks-mutation-operations). The generated mutation functions follow the naming convention of SDK functions and by default append `Mutation`, e.g., `addPetMutation()`.

<Tabs>
  <TabItem label="example">
```ts title="colada.gen.ts"
const addPet = useMutation({
  ...addPetMutation(),
  onError: (error) => {
    console.log(error);
  },
});

addPet.mutate({
  body: {
    name: 'Kitty',
  },
});
```
  </TabItem>
  <TabItem label="config">
```js {8} title="openapi-ts.config.ts"
export default {
  input: 'hey-api/backend', // sign up at app.heyapi.dev
  output: 'src/client',
  plugins: [
    // ...other plugins
    {
      name: '@pinia/colada',
      mutationOptions: true,
    },
  ],
};
```
  </TabItem>
</Tabs>

You can customize the naming and casing pattern for `mutationOptions` functions using the `.name` and `.case` options.
