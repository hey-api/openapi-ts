# Web Frameworks

There are two approaches to developing APIs: code-first, where you start with the code, or spec-first, where you begin with the specification. If you use the latter, you can ensure your APIs adhere to the specification with our web framework plugins.

## Options

Hey API natively supports the following frameworks.

- [Angular](/docs/openapi/typescript/plugins/angular)
- [Fastify](/docs/openapi/typescript/plugins/fastify)
- [Nest](/docs/openapi/typescript/plugins/nest)
- [oRPC](/docs/openapi/typescript/plugins/orpc)
- [Adonis](/docs/openapi/typescript/plugins/adonis) <Badge text="Vote" variant="tip" />
- [Elysia](/docs/openapi/typescript/plugins/elysia) <Badge text="Vote" variant="tip" />
- [Express](/docs/openapi/typescript/plugins/express) <Badge text="Vote" variant="tip" />
- [Hono](/docs/openapi/typescript/plugins/hono) <Badge text="Vote" variant="tip" />
- [Koa](/docs/openapi/typescript/plugins/koa) <Badge text="Vote" variant="tip" />

Don't see your framework? Let us know by [opening an issue](https://github.com/hey-api/openapi-ts/issues).
